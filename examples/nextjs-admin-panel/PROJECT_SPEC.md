# Next.js Admin Application - Detailed Specification

## Project Overview

A comprehensive admin dashboard for managing and monitoring the product scraper ecosystem, including categories, products, brands, ingredients, and scraping queues across multiple retailers.

---

## Tech Stack

### Core
- **Framework**: Next.js 14+ (App Router)
- **Database**: PostgreSQL via Prisma
- **UI Framework**: React 18+
- **Styling**: Tailwind CSS + shadcn/ui components
- **State Management**: React Query (TanStack Query) for server state
- **Forms**: React Hook Form + Zod validation
- **Tables**: TanStack Table v8 (formerly React Table)
- **Authentication**: NextAuth.js (using existing account_user table)

### Advanced Features
- **Search**: Full-text search with PostgreSQL + debounced search
- **Filtering**: Multi-level filter system with URL state persistence
- **Data Visualization**: Recharts for analytics
- **Export**: CSV/Excel export capabilities
- **Real-time Updates**: SWR or React Query polling for queue monitoring
- **Bulk Operations**: Multi-select with batch updates

### Bulk Operations
- **Enable/Disable Selected** - Toggle `enabled` flag
- **Mark Products Done** - Set `is_category_url_product_done = true`
- **Reset Queue Status** - Reset to "pending"
- **Delete Selected** - Soft delete (with confirmation)
- **Export Selected** - Export to CSV/Excel

### URL State Management

```typescript
// Persist filters in URL for shareable links
const searchParams = useSearchParams();
const router = useRouter();

const updateFilters = (newFilters) => {
  const params = new URLSearchParams(searchParams);
  Object.entries(newFilters).forEach(([key, value]) => {
    if (value) params.set(key, value);
    else params.delete(key);
  });
  router.push(`?${params.toString()}`);
};
```
